# Delivery Checklist

## Phase 1 — Prepare
- Read the brief
- Create your private repo
- Create a feature branch: `feat/stock-snapshot`

## Phase 2 — Design
- Fill `templates/analysis-template.md`
- Create a Mermaid diagram using `templates/mermaid-diagram-template.md`
- Commit design docs

## Phase 3 — Build
- Implement CDS (Interface + Consumption preferred)
- Implement Low Stock logic
- Create Service Definition + Binding
- Validate your service (preview/demo evidence)

## Phase 4 — Evidence
- Fill `task/EVIDENCE.md`
- Open PR into `main`
- Submit repo URL + commit hash in SourcAP
